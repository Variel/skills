# 시각적 완성도 + 검증 + 안티패턴

## Part VI. 시각적 완성도 — 디테일의 기술

### 1. 미학적 방향의 의도성

디자인 시작 전에 **대담한 미학적 방향**을 선택하고 정밀하게 실행하라. 의도성이 강도보다 중요하다.

가능한 방향들(이것은 출발점이지 전부가 아니다 — 맥락에 맞는 독자적 방향을 만들 수도 있다):
- 극도의 미니멀리즘 — 빼기의 미학, 완벽한 여백
- 맥시멀리스트 — 겹겹의 레이어, 풍부한 텍스처, 제한 없는 표현
- 레트로-퓨처리스틱 — 과거의 미래 상상, 네온과 CRT
- 유기적/자연적 — 곡선, 자연 색, 손으로 만든 느낌
- 럭셔리/세련됨 — 클래식 타이포, 절제된 색상, 여유로운 간격
- 놀이적/장난감 같은 — 둥근 모서리, 밝은 색, 바운스 애니메이션
- 에디토리얼/매거진 — 대담한 타이포 대비, 그리드 파괴, 비대칭
- 브루탈리스트/날것 — 시스템 폰트, 날 것의 구조, 장식 거부
- 아르데코/기하학 — 대칭, 금속 액센트, 기하학 패턴
- 소프트/파스텔 — 부드러운 색, 글래스모피즘, 부유하는 느낌
- 산업적/실용적 — 모노스페이스, 고밀도, 기능 우선
- 테크노/글리치 — 디지털 결함, 왜곡, 반복 패턴
- 뉴모피즘 — 부드러운 그림자, 볼록/오목, 만질 수 있는 느낌

**핵심은 의도성(intentionality)이지 강도(intensity)가 아니다.** 선택한 방향을 끝까지 밀고 나가라.

**차별화 질문**: 이 디자인에서 사람들이 기억할 **단 하나**는 무엇인가? 이 질문에 답하지 못하면, 방향을 더 날카롭게 가다듬어라.

**복잡도 매칭 원칙**: 미학적 비전의 강도에 맞춰 구현 복잡도를 조절하라.
- 맥시멀 비전 → 정교한 애니메이션, 풍부한 레이어, 텍스처, 그라디언트 메시
- 미니멀 비전 → 극도의 정밀함, 완벽한 간격, 미세한 호버 변화, 우아한 전환
- **양쪽 모두 같은 양의 노력을 요구한다. 미니멀이 쉽다는 착각을 경계하라.**

### 2. 타이포그래피

- **독특하고 아름다운 폰트를 선택하라.** Arial, Inter, Roboto, system-ui 같은 범용 폰트를 피하라
- 특징적인 디스플레이 폰트 + 세련된 본문 폰트 조합
- 매번 다른 조합 시도 — 같은 폰트에 수렴하지 말라 (Space Grotesk 같은 AI 인기 폰트도 피하라)
- 폰트 페어링은 **대비**에서 온다: 세리프 + 산세리프, 기하학 + 유기적, 굵은 + 가벼운
- 한글 프로젝트: Pretendard, SUIT 같은 오픈소스 한글 폰트도 항상 같은 것만 쓰지 말라 — Noto Sans KR, Spoqa Han Sans Neo, IBM Plex Sans KR 등 다양하게 시도하라

### 3. 색상과 테마

- CSS 변수로 일관성 확보
- **지배적 색상 + 날카로운 강조색** > 소심하게 균등 분배된 팔레트
- 시맨틱 색상을 목적에 맞게 (에러=에러색, 성공=성공색)
- 하드코딩된 hex/rgb 대신 변수/토큰 사용

#### 디자인 토큰 참고

| 토큰 | 용도 |
|------|------|
| `background` | 페이지 배경 |
| `foreground` | 주 텍스트 |
| `muted-foreground` | 보조 텍스트, 플레이스홀더 |
| `card` | 카드 배경 |
| `border` | 테두리, 구분선 |
| `primary` | 주 행동, 브랜드 |
| `destructive` | 위험 행동 |

### 4. 모션과 인터랙션

**임팩트 우선순위**: 잘 짜인 페이지 로드 애니메이션(엇갈린 등장, animation-delay) 하나가 흩어진 마이크로인터랙션 열 개보다 더 강력하다.

구현 전략:
- **CSS-only 우선**: `@keyframes`, `transition`, `animation-delay`로 가능한 것부터
- **React**: Motion(framer-motion) 라이브러리 사용 가능 시 활용
- **스크롤 인터랙션**: Intersection Observer로 뷰포트 진입 시 트리거. threshold 0.2, rootMargin "-10%"
- **호버 상태**: 예상치 못한 변화를 넣어라 — 단순한 opacity 변화를 넘어서

핵심 모션 패턴:
```
페이지 로드 → staggered reveal (각 요소 100-150ms 간격)
스크롤 진입 → fade-in + translateY(20px→0), 600ms ease-out
호버 → scale, 색상 전환, 그림자 변화 (150ms)
상태 전환 → spring 애니메이션 (자연스러움)
```

- **맥시멀 비전** → 정교한 다단계 애니메이션, 패럴랙스, 3D 변환
- **미니멀 비전** → 절제된 전환, 미세한 이징 차이, 정밀한 타이밍
- **prefers-reduced-motion**: 반드시 존중하라 — 대안으로 즉시 전환 또는 페이드만 제공

### 5. 공간 구성과 배경

**공간 자체가 디자인 언어다.** 단색 배경에 안주하지 말라 — 분위기를 만들어라.

배경 & 텍스처 도구상자:
- **그라디언트 메시**: 다중 색상이 유기적으로 혼합되는 배경
- **노이즈 텍스처**: SVG 필터 또는 CSS background-image로 미세한 질감
- **기하학 패턴**: 반복되는 도형으로 리듬과 깊이
- **그레인 오버레이**: 필름 그레인 효과로 유기적 느낌 (`mix-blend-mode: overlay`)
- **레이어드 투명도**: 겹쳐진 반투명 요소로 깊이감
- **극적인 그림자**: `box-shadow` 다중 레이어로 부유 효과
- **커스텀 커서**: 맥락에 맞는 커서 변경으로 인터랙션 힌트
- **장식적 테두리**: 점선, 이중선, 그라디언트 보더로 구역 강조

공간 구성 전략:
- **비대칭**: 의도적 불균형으로 시선 유도
- **겹침(Overlap)**: 요소를 겹쳐 깊이와 관계 표현
- **대각선 흐름**: 수평/수직 그리드를 깨는 사선 배치
- **그리드 파괴**: 의도적으로 그리드를 이탈하는 요소 하나가 전체를 살린다
- **넉넉한 여백 또는 통제된 밀도**: 극단을 선택하되 중간에서 어정쩡하지 말라

### 6. 이미지 사용 위계

| 순위 | 유형 | 설명 |
|------|------|------|
| 1 | **전환 이미지** | After 상태의 사람들. 제품은 없거나 주변부 |
| 2 | **맥락적 사용** | 실제 환경에서 제품 사용하는 사람들 |
| 3 | **환경 속 제품** | 사용/결과를 암시하는 환경에 놓인 제품 |
| 4 | **고립된 제품** | 제품 단독. 최소한으로 |

**모든 이미지는 방문자의 미래 삶에서 온 한 장면이어야 한다.**

## Part VII. 검증 — 디자인이 제대로 작동하는가

### 1. 자기 점검 체크리스트

#### 사용자 관점

- [ ] 처음 방문한 사용자가 3초 안에 화면 목적을 이해할 수 있는가?
- [ ] 다음에 해야 할 행동이 명백한가?
- [ ] 사용자가 추측해야 하는 부분이 있는가? → 재설계
- [ ] 모든 상태(로딩, 빈, 에러, 성공)가 고려되었는가?
- [ ] 모든 행동에 피드백이 있는가?

#### 구조 관점

- [ ] 화면에 하나의 지배적 목적이 있는가?
- [ ] 하나의 지배적 시각 영역이 있는가?
- [ ] 정보 위계가 명확한가?
- [ ] 밀도가 의도적인가?
- [ ] 불필요한 요소가 있지는 않은가?

#### 일관성 관점

- [ ] 같은 문제를 같은 방식으로 해결하고 있는가?
- [ ] 간격이 일관된 척도를 따르는가?
- [ ] 색상이 시맨틱 토큰을 사용하는가?
- [ ] 타이포그래피 체계가 일관적인가?

#### 확장성 관점

- [ ] 데이터가 10배 늘어나도 깨지지 않는가?
- [ ] 항목이 추가되어도 구조가 유지되는가?
- [ ] 다른 브레이크포인트에서도 위계가 살아있는가?

### 2. 콘텐츠 우선 검증

시각 디자인을 제거하고 콘텐츠만 남겼을 때:
- 가치가 몇 초 안에 명확한가?
- 흐름이 논리적인가?
- 이점이 결과 중심인가?
- 신뢰가 충분한가?
- **비주얼 없이도 페이지가 작동하는가?**

콘텐츠가 이 테스트를 통과해야 비주얼 작업을 시작할 수 있다.

### 3. 미학적 안티패턴 (AI 디자인에서 특히 경계하라)

이 목록에 해당하는 선택을 했다면, 의식적으로 다시 생각하라:

| 안티패턴 | 왜 문제인가 | 대안 |
|---------|------------|------|
| Inter, Roboto, Arial, system-ui 반복 | 개성 부재, AI 냄새 | 매번 다른 독특한 폰트 |
| Space Grotesk 매번 사용 | AI 생성물의 대표 클리셰 | 다른 기하학 산세리프 탐색 |
| 보라색 그라디언트 + 흰 배경 | 가장 흔한 AI 디자인 패턴 | 대담한 색상 조합 |
| 모든 요소 둥근 모서리 16px | 차별화 없는 "안전한" 선택 | 맥락에 맞는 radius 선택 |
| 예측 가능한 카드 그리드 | 모든 AI 출력이 비슷해짐 | 비대칭, 겹침, 그리드 파괴 |
| 단색 배경에 그림자 카드 | 깊이 없는 평면 | 텍스처, 그라디언트, 패턴 |
| 동일한 hover opacity 변화 | 인터랙션에 개성 없음 | scale, 색상, 그림자, 변형 |
| 매번 다크 모드 기본 | 패턴 수렴 | 라이트/다크/특수 테마 교대 |

### 4. 주기적 시각 검증

디자인 작업 중 주기적으로 스크린샷 확인:
- 생성된 결과가 요구사항을 충족하는가?
- 시각적 오류, 결함, 정렬 불일치가 없는가?
- 의도한 레이아웃과 일치하는가?
- 컴포넌트가 올바르게 구성되었는가?
- **섹션 0의 "차별화" 질문에 답하는 디자인인가?**
- **AI 클리셰에 빠지지 않았는가?** (위의 안티패턴 테이블 확인)
