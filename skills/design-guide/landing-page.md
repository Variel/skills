# 랜딩 페이지 디자인 — 플랫폼 개요

> 이 문서는 랜딩 페이지 디자인 작업 시 **반드시 먼저 읽는 진입점**이다. 세부 가이드로 바로 가지 말고, 여기서 방향을 잡은 뒤 필요한 문서를 순서대로 참조하라.

---

## 랜딩 페이지란

하나의 목표를 위해 존재하는 단일 페이지. 방문자를 특정 행동(가입, 구매, 데모 요청)으로 유도하거나, 브랜드/제품의 가치를 전달한다. 앱이 아니라 **설득의 도구**다.

---

## 설계 워크플로우 — 이 순서를 따르라

### Step 1. 패러다임 선택 (필수, 최우선)

`landing-page-paradigms.md`를 읽고 **페이지의 지배적 패러다임을 하나 선택하라.**

| 패러다임 | 핵심 전제 | 대표 사례 |
|---------|----------|----------|
| **전환 엔진** | 모든 요소가 하나의 CTA를 향한다. 퍼널 구조 | Dropbox, Mailchimp |
| **내러티브 스크롤** | 스토리 드리븐. 스크롤 = 장면 전환. 감정 여정 | Apple, Porsche |
| **제품-주도** | 직접 체험이 설득. 데모, 플레이그라운드, 코드 샘플 | Vercel, Supabase |

> **판단 기준**: "방문자에게 가장 효과적인 설득 방식은?"
> - 빠른 전환이 목표 → 전환 엔진
> - 브랜드 인상과 감성 전달이 우선 → 내러티브 스크롤
> - 제품 자체가 최고의 영업사원 → 제품-주도

### Step 2. 섹션별 설계 (구현 시)

`landing-page-sections.md`를 읽고 페이지를 구성하는 각 섹션의 설계 규칙을 따르라.

| 섹션 | 언제 참조하는가 |
|-----|---------------|
| 헤더/내비게이션 | 상단 고정 바, 메뉴 구성 시 |
| 히어로 (5가지 변형) | 페이지 첫 화면 설계 시 |
| 신뢰/소셜 프루프 | 로고 바, 통계, 후기, 케이스 스터디 배치 시 |
| 기능 섹션 (4가지 변형) | 제품 기능을 소개할 때 |
| "어떻게 작동하는가" | 3~4단계 프로세스 설명 시 |
| 가격 | 플랜 카드, 비교표, 토글 설계 시 |
| FAQ | 자주 묻는 질문 섹션 시 |
| 최종 CTA | 페이지 하단 마무리 전환 유도 시 |
| 푸터 | 링크 칼럼, 소셜, 뉴스레터 시 |

### Step 3. 실무 수치 확인 (세부 구현 시)

`landing-page-practical.md`를 읽고 구체적인 수치와 최적화 기법을 적용하라.

| 주제 | 언제 참조하는가 |
|-----|---------------|
| 마이크로카피 | 버튼 텍스트, 폼 레이블, CTA 문구 작성 시 |
| 성능 (Core Web Vitals) | LCP, INP, CLS 최적화 시 |
| 모바일 반응형 | 레이아웃 전환, 타이포 스케일링, 모바일 CTA 시 |
| 업종별 변형 | SaaS, 이커머스, 에이전시, 이벤트, 오픈소스 등 |
| 간격 체계 (4가지 접근법) | 섹션 간 여백과 밀도를 결정할 때 |
| 스크롤 인터랙션 | Intersection Observer, 스티키, 패럴랙스 시 |
| SEO/메타데이터 | 시맨틱 HTML, 구조화 데이터, alt 텍스트 시 |

### Step 4. 마감 검증 (필수, 최종)

`visual-and-verification.md`를 읽고 미학적 방향과 검증 체크리스트를 적용하라. 플랫폼과 무관하게 모든 디자인의 마감 단계에서 반드시 거쳐야 한다.

---

## 설계 전 필수 확인 (Hard Gate)

랜딩 페이지는 다른 플랫폼과 달리, **설계 전에 반드시 다음을 확정해야 한다.** 답이 불명확하면 설계를 시작하지 말라:

- **제품**: 무엇인가, 어떤 문제를 해결하는가
- **대상**: 누구를 위한 것인가
- **목표**: 주된 전환 행동, 보조 목표
- **가치 제안**: 경쟁 대비 무엇이 다른가/나은가
- **톤**: 브랜드 성격, 제약, 피할 단어
- **콘텐츠 제약**: 필수/금지 섹션, 법적 요구
- **시각 자료**: 브랜드 색상, UI 스크린샷, 로고
