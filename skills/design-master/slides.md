# 프레젠테이션 슬라이드 디자인 가이드

> 프레젠테이션의 본질적 질문: "이 덱은 스피커를 위한 것인가, 아니면 문서 대용인가?" 그 답에 따라 설계 철학이 완전히 갈린다.

## 패러다임 A: 스피커 보조물 — 슬라이드는 배경이다

"청중은 슬라이드를 읽는 것이 아니라, 스피커의 말을 듣는다."

핵심 사상: **슬라이드에서 텍스트를 제거할수록, 스피커의 메시지가 강해진다.**

```
우선순위: 명확성 > 가독성 > 위계 > 단순함
```

설계 규칙:
- **슬라이드당 하나의 아이디어** (순수 비주얼도 허용)
- 내용이 맞지 않으면 **분할 or 제거. 절대 폰트를 줄이지 말라**
- 일관성 > 창의성
- 차트 > 텍스트. 차트당 하나의 인사이트
- 첫/마지막 슬라이드 = **감정적 선언**

타이포그래피:
- 최대 2개 폰트 패밀리
- **최소 28pt** / 본문 36-80pt / 타이틀 80-200pt
- 크기가 아니라 두께(weight)로 구분
- line-height ~1.1-1.2 / **항상 높은 대비**

포맷:
- 16:9, 1920×1080
- 콘텐츠 사방 100px+ 여백

**적합**: 컨퍼런스 기조연설, TED, 제품 발표, 대규모 행사.

---

## 패러다임 B: 자립형 문서 — 슬라이드가 스스로 말한다

스피커 없이도 메시지 전달. "덱이 이메일로 전달되거나, 사후에 혼자 읽힌다."

핵심 사상: **모든 슬라이드는 자기 완결적. 타이틀이 곧 결론, 본문이 근거.**

```
참조: McKinsey, Amazon 6-pager 슬라이드 버전

작동 원리:
  1. 타이틀 = "액션 타이틀" (결론 문장)
     나쁜 예: "매출 현황"
     좋은 예: "3분기 매출이 전년 대비 23% 증가하며 목표를 초과 달성했다"
  2. 본문 = 타이틀의 근거 (차트, 표, 불릿)
  3. 출처/주석 = 하단 작은 글씨

슬라이드 구조:
  ┌────────────────────────────────────────────┐
  │  [액션 타이틀: 결론 문장, 28-32pt Bold]      │  상단 20%
  ├────────────────────────────────────────────┤
  │      근거 영역 (차트, 표, 다이어그램)        │  중앙 65%
  │      텍스트: 18-24pt                        │
  ├────────────────────────────────────────────┤
  │  출처: 14pt / 페이지 번호                    │  하단 15%
  └────────────────────────────────────────────┘
```

설계 규칙:
- 타이틀만 읽어도 전체 덱 논리가 이어져야 한다 ("타이틀 스킴 테스트")
- 텍스트 밀도 높을 수 있다: 18pt까지 허용
- 차트에 핵심 수치 하이라이트 필수
- 하단에 출처/날짜/페이지 번호 항상 표기

**적합**: 경영 보고, 이사회, 사내 전략, 이메일 첨부용.

---

## 패러다임 C: 데이터 저널리즘 — 숫자가 이야기를 한다

데이터 시각화가 중심. "데이터를 올바르게 시각화하면 설득은 자동으로 따라온다."

핵심 사상: **차트 하나, 인사이트 하나. 데이터가 결론을 말하게.**

```
참조: Hans Rosling, The Economist

슬라이드 구조:
  ┌────────────────────────────────────────────┐
  │  인사이트 헤드라인 (28-36pt, Bold)           │
  ├────────────────────────────────────────────┤
  │              차트 / 그래프                   │  60-70%
  │      ── 핵심 데이터포인트 하이라이트 ──      │
  ├────────────────────────────────────────────┤
  │  맥락 한 줄 (20pt) + 출처 (14pt)            │
  └────────────────────────────────────────────┘

차트 원칙:
  - Chart Junk 제거: 배경 격자선, 3D 효과, 장식 없음
  - 데이터-잉크 비율 극대화 (Edward Tufte)
  - 핵심 데이터포인트 강조색, 나머지 회색
  - 범례를 데이터 옆에 직접 라벨링
  - Y축 0에서 시작 (막대), 절삭 가능 (꺾은선)
  - 애니메이션: 데이터 빌드업 (연도별 순차)
```

설계 규칙:
- 슬라이드당 차트 1개 (비교 시 최대 2개)
- 수치 단위 명확: "23%" → "매출 23% 증가 (YoY)"
- 색상 = 의미: 긍정=초록, 부정=빨강, 중립=회색
- Before/After 동일 축척 유지
- 데이터 없는 슬라이드 전체의 20% 이내

**적합**: 분기 리뷰, 시장 분석, 투자 피치, 연구 발표.

---

> **같은 주제, 다른 패러다임**: "분기 실적" — CEO 전사 발표 = 스피커 보조물, CFO 이사회 = 자립형 문서, 데이터팀 분석 = 데이터 저널리즘. 청중과 전달 맥락에 따라 선택.

## 실무 수치 — 타이포, 레이아웃, 색상

### 접근법 A: 기업 프레젠테이션 — 구조/데이터 중심

McKinsey, Bain 스타일.

```
캔버스: 1920×1080px (16:9)
안전 영역: 사방 80px+

타이포:
  액션 타이틀 = 28-36pt Bold (결론 자체)
  본문        = 20-24pt Regular / line-height 1.3
  주석/출처   = 14-16pt Light / #999999
  데이터 숫자 = 48-72pt Bold

색상 (보수적):
  배경 #FFFFFF / 교대 #F8F9FA
  주 텍스트 #1A1A2E
  강조 #2563EB (파랑)
  보조 #059669 (초록)
  경고 #DC2626

레이아웃:
  타이틀 영역 = 80-160px
  콘텐츠 영역 = 160-960px
  하단 여백   = 960-1080px
  칼럼 간 간격 = 32-48px
```

### 접근법 B: 스타트업 피치덱 — 대담한 타이포

YC, Sequoia 스타일.

```
타이포:
  히어로 숫자   = 120-200pt / weight 900 / tracking -0.04em
  히어로 텍스트 = 64-96pt / weight 800 / 최대 2줄
  서브 텍스트   = 28-36pt / weight 400
  라벨          = 14-18pt / 600 / uppercase / tracking +2px

색상 (대담):
  어두운 배경 #0F172A / 밝은 #FFFFFF
  텍스트 #F8FAFC (다크) / #0F172A (라이트)
  강조 = 단일 네온 컬러 (시안 #06B6D4 / 오렌지 #F97316 / 보라 #8B5CF6)

콘텐츠 밀도:
  커버    = 로고 + 태그라인 + 설립연도 (3요소)
  문제    = 숫자 하나 + 한 문장
  솔루션  = 스크린샷 하나 + 한 줄
  트랙션  = 우상향 그래프 하나
  Ask     = 금액 + 용도 3줄 이내
```

### 접근법 C: 키노트/컨퍼런스 — 시네마틱

Apple Keynote, TED 스타일.

```
타이포:
  키워드   = 80-160pt / weight Black / 1-3단어만
  보조     = 36-48pt / weight Light
  텍스트 없는 슬라이드 = 전체의 30-50%

이미지:
  풀블리드     = 1920×1080, gradient overlay
  반분할       = 960×1080 이미지 + 960×1080 텍스트
  배경 블러    = blur(40px) + rgba(0,0,0,0.5)

전환:
  슬라이드 간 = fade 0.5s / slide-left 0.3s
  요소 등장   = stagger 0.1s 간격

색상: 다크 배경 #000000, 텍스트 #FFFFFF, 강조 = 브랜드 컬러 1개
```

### 접근법 D: 워크숍/교육 — 정보 전달

Coursera, Google I/O 스타일.

```
타이포:
  섹션 번호      = 120pt / weight 900 / 강조색
  슬라이드 타이틀 = 36-44pt Bold
  본문           = 24-28pt Regular / line-height 1.4
  코드           = 20-24pt monospace / 배경 #1E1E1E / 패딩 32px

레이아웃:
  개념 설명 = 좌 텍스트 50% + 우 다이어그램 50%
  코드 데모 = 상단 코드 60% + 하단 결과 40%
  비교      = Before(#FEE2E2) / After(#DCFCE7)
  퀴즈      = 중앙 "?" + 선택지

색상:
  밝은 배경 #FFFFFF / 코드 #1E293B
  텍스트 #334155
  코드 하이라이트 rgba(250,204,21,0.3)
  완료 #22C55E / 미완료 #CBD5E1

진행 표시: 상단 4px 프로그래스 바 또는 "3/12" 카운터
```
