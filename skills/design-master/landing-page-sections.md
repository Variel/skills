# 랜딩 페이지 디자인 가이드 — 섹션별 설계 패턴

"레이아웃 잡고, 색상 정하라"에서 멈추면 모든 랜딩 페이지가 동일해진다. 아래는 섹션마다 반복적으로 등장하는 결정들을 구체적으로 정리한 것이다.

## 헤더/내비게이션

```
기본 구조:
  ┌─────────────────────────────────────────────────┐
  │ [로고]   기능 | 가격 | 문서 | 블로그    [로그인] [CTA] │
  └─────────────────────────────────────────────────┘

높이: 56-72px
위치: sticky top (스크롤 시 고정)
배경: 투명 → 스크롤 후 blur(12px) + 배경색 (backdrop-filter)
z-index: 50 (모든 콘텐츠 위)
좌우 패딩: 24-48px (max-width와 동일)

네비게이션 항목:
  개수: 4-6개 (초과 시 "더보기" 드롭다운)
  폰트: 14-15px, weight 500
  활성: primary 색 또는 underline
  호버: opacity 0.7 → 1.0

CTA 버튼:
  헤더 내 CTA: 히어로 CTA와 동일한 행동 (일관성)
  크기: 다른 항목보다 시각적으로 돋보이게 (filled 버튼)

모바일:
  ┌──────────────────────┐
  │ [로고]        [☰] [CTA] │
  └──────────────────────┘

  햄버거 메뉴: 전체 화면 오버레이 또는 우측 슬라이드인
  CTA 버튼: 모바일에서도 헤더에 유지 (축약 가능: "시작하기" → 아이콘)
  안티패턴: 모바일에서 내비게이션을 숨기고 CTA도 숨기는 것

스크롤 반응:
  아래로 스크롤: 헤더 숨김 (transform: translateY(-100%))
  위로 스크롤: 헤더 재등장
  전환: 300ms ease-in-out
  대안: 항상 표시 (짧은 페이지) 또는 미니 헤더 (높이 축소)
```

## 히어로 섹션 변형

히어로는 랜딩 페이지의 "첫인상"이다. 방문자의 70%가 히어로만 보고 떠나거나 스크롤한다. **히어로에 모든 것을 거라.**

```
변형 A — 중앙 정렬 (Stripe, Linear):
  ┌────────────────────────────────────────────┐
  │              [배지: New]                     │
  │     가치 제안 헤드라인 (중앙)                  │
  │     부제목 설명 (중앙, max-width: 640px)      │
  │     [주 CTA]  [보조 CTA]                    │
  │                                            │
  │     [제품 스크린샷/비주얼]                     │
  │                                            │
  │     신뢰 로고 바                              │
  └────────────────────────────────────────────┘

  장점: 깔끔, 집중력 극대화
  단점: 비주얼이 약하면 밋밋
  적합: B2B SaaS, 개발자 도구, API

변형 B — 2칼럼 분할 (HubSpot, Shopify):
  ┌────────────────────┬───────────────────────┐
  │  배지               │                       │
  │  헤드라인            │   [제품 스크린샷]       │
  │  서브라인            │   또는 [비디오]         │
  │  [CTA]  [보조CTA]  │                       │
  │  신뢰 로고 바        │                       │
  └────────────────────┴───────────────────────┘

  비율: 텍스트 50-55% : 비주얼 45-50%, gap 48-64px
  장점: 텍스트 + 비주얼 동시 전달
  단점: 모바일에서 쌓이면 비주얼이 폴드 아래로
  적합: PLG 제품, 이커머스, 마케팅 도구

변형 C — 풀스크린 비주얼 (Apple, Nothing):
  ┌────────────────────────────────────────────┐
  │                                            │
  │        [배경: 풀블리드 이미지/비디오]          │
  │                                            │
  │            헤드라인 (오버레이)                 │
  │            [CTA]                            │
  │                                            │
  └────────────────────────────────────────────┘

  오버레이: gradient rgba(0,0,0,0.3-0.6)
  텍스트: 중앙 또는 좌하단
  장점: 감성적 임팩트 극대화
  단점: 정보 밀도 최소 (텍스트 2줄 한계)
  적합: 하드웨어, 럭셔리, 이벤트, 크리에이티브 도구

변형 D — 비디오 히어로:
  배경 비디오: 자동 재생, 무음, 루프
  포맷: MP4 (H.264), 해상도 1920×1080, 용량 5MB 이하
  대체: poster 이미지 (비디오 로드 전)
  모바일: 비디오 대신 정적 이미지 (데이터 절약)

  규칙: 비디오가 텍스트 가독성을 방해하면 안 됨 (오버레이 필수)
  규칙: 사용자가 비디오를 멈출 수 있어야 함 (접근성)

변형 E — 인터랙티브 히어로:
  타이핑 애니메이션 헤드라인 ("_를 위한 도구" → 순환)
  3D 모델/WebGL 배경 (Three.js)
  마우스 추적 파티클/그라디언트
  인라인 데모 위젯

  규칙: 인터랙션이 핵심 메시지를 가리면 안 됨
  규칙: prefers-reduced-motion에서 정적 버전 제공
  규칙: 모바일에서 성능 확인 필수 (GPU 가속 테스트)

히어로 배지(Announcement Badge):
  "✨ 새로운 기능 출시" / "Product Hunt #1" / "Y Combinator W24"
  크기: 12px/600, padding 4-6px 12-16px, radius 99px
  색상: 연한 primary 배경(primary/10%) + primary 텍스트
  위치: 헤드라인 바로 위, 12-16px 간격
  링크: 클릭 시 상세 페이지 또는 블로그로 이동
```

## 신뢰 & 사회적 증명

사회적 증명 없는 랜딩 페이지는 **"우리가 좋다고 하니까 좋다"**와 같다. 방문자는 당신의 말보다 다른 사용자의 경험을 믿는다.

```
로고 바 (Logo Cloud):
  위치: 히어로 바로 아래 또는 히어로 내부 하단
  레이블: "이 기업들이 사용합니다" 또는 "Trusted by"
  로고: 5-8개, 높이 24-32px, 회색조(grayscale)
  호버: 원래 색상으로 전환 (선택적)
  간격: 로고 간 32-48px
  정렬: 수평 중앙 또는 좌측 정렬
  모바일: 2행으로 축소하거나 수평 스크롤

  안티패턴: 알려지지 않은 로고만 나열 (신뢰 오히려 감소)
  안티패턴: 너무 많은 로고 (10개+ → 공간 낭비, 인지 과부하)

수치 증명 (Stats Bar):
  구조: 3-4개 수치 + 레이블을 수평 배치
  예시:
    10,000+        99.9%        <50ms        150+
    활성 사용자     가동률       응답 시간     국가
  수치: 32-48px, weight 700-800
  레이블: 14px, muted-foreground
  구분: 세로선 1px 또는 여백만으로 구분
  애니메이션: 뷰포트 진입 시 카운트업 (0 → 10,000+ / 1.5초)

  안티패턴: 무의미한 수치 ("1억 다운로드"가 개인 도구에서 무슨 의미인가?)
  안티패턴: 반올림이 과한 숫자 (523→500+ 는 OK, 12→10+는 인위적)

추천사(Testimonial):
  유형 A — 카드 그리드:
    ┌──────────────────┐
    │ "제품 덕분에..."  │
    │                  │
    │ 😊 이름, 직함     │
    │ @회사 로고        │
    └──────────────────┘
    2-3칼럼 그리드, 카드당 max 3줄 인용

  유형 B — 히어로 추천사:
    풀 너비, 큰 따옴표("), 인용문 20-28px italic
    하단: 사진(48-64px 원형) + 이름 + 직함

  유형 C — 비디오 추천사:
    가장 강력한 형태. 썸네일 + 재생 버튼
    길이: 30-90초. 짧을수록 좋다

  유형 D — 트위터/소셜 임베드:
    실제 트윗/포스트 스크린샷 또는 임베드
    신뢰도 높음 (수정 불가하므로)
    정리: masonry 그리드

  규칙: 이름 + 직함 + 회사 로고 + 얼굴 사진 = 최대 신뢰
  규칙: 사진 없는 추천사는 신뢰도 30% 하락
  규칙: 구체적인 결과("매출 23% 증가") > 모호한 칭찬("정말 좋습니다")

사례 연구(Case Study):
  구조: 로고 + 결과 수치(큰 폰트) + 한 줄 설명 + [사례 보기] 링크
  배치: 2-3칼럼 카드 또는 수직 스택
  적합: B2B, 엔터프라이즈 랜딩 페이지

미디어 노출(Press/Awards):
  "Featured in" 섹션: 미디어 로고 나열
  배치: 로고 바와 동일한 형식
  적합: 초기 스타트업 (YC, TechCrunch, Product Hunt 뱃지)
```

## 기능 섹션

```
기능 소개 패턴:

패턴 A — 수직 스택 (Stripe 스타일):
  각 기능이 전체 너비, 좌우 교대 배치
  ┌────────────────────────────────────────────┐
  │  기능 1 제목        [스크린샷]               │
  │  설명 텍스트                                │
  ├────────────────────────────────────────────┤
  │  [스크린샷]           기능 2 제목            │
  │                     설명 텍스트              │
  └────────────────────────────────────────────┘
  좌/우 교대로 시선 흐름 유지
  각 섹션 padding: 80-120px (상하)
  적합: 핵심 기능 3-5개 (각각 상세 설명 필요 시)

패턴 B — 카드 그리드 (Notion 스타일):
  3-4칼럼 카드, 아이콘 + 제목 + 설명
  ┌──────────┐ ┌──────────┐ ┌──────────┐
  │ 🔒       │ │ ⚡       │ │ 🎨       │
  │ 보안      │ │ 속도     │ │ 디자인    │
  │ 설명...   │ │ 설명...  │ │ 설명...   │
  └──────────┘ └──────────┘ └──────────┘
  카드 padding: 24-32px
  아이콘: 32-48px (primary 색 또는 그라디언트)
  적합: 보조 기능 6-12개 (간략 소개)

패턴 C — 탭 전환 (Linear 스타일):
  상단: 기능 탭 바 (수평)
  하단: 선택된 기능의 스크린샷 + 설명
  전환: fade 200ms 또는 slide
  적합: 기능이 시각적으로 다른 화면일 때

패턴 D — 스크롤 연동 (Apple 스타일):
  좌측: 기능 목록 (sticky)
  우측: 스크롤에 따라 스크린샷 전환
  또는: sticky 이미지 + 옆 텍스트가 스크롤에 따라 교체

  구현: Intersection Observer로 활성 기능 감지
  적합: 기능마다 비주얼이 강한 제품

기능 설명 카피 규칙:
  제목: 결과 중심 ("파일 관리" → "파일을 잃어버리지 않습니다")
  설명: 2-3줄 이내
  강조: Bold로 핵심 문구 하이라이트
  안티패턴: 전문 용어 나열 ("GraphQL 기반 실시간 동기화" → "팀원과 동시에 작업하세요")
```

## "작동 방식" (How It Works)

```
구조: 3-4단계 프로세스
  ① → ② → ③

표현:
  수평 카드: 3칼럼, 번호 + 아이콘 + 제목 + 설명
  수직 타임라인: 좌측 번호/아이콘, 우측 설명, 연결선
  일러스트: 각 단계가 하나의 이미지 scene

  번호: 48-64px, weight 700-900, primary 색 또는 muted
  아이콘: 번호 대신 사용 시 48px
  연결선: 번호 사이 1-2px dashed, muted 색

규칙:
  3단계가 이상적 (인지 부하 최소)
  각 단계: 동사로 시작 ("가입하세요", "설정하세요", "결과를 확인하세요")
  마지막 단계: 결과/성취 느낌 ("성장을 경험하세요")

  안티패턴: 5단계 이상 (3단계로 재구성하라)
  안티패턴: 실제 사용 흐름과 다른 이상화된 단계
```

## 가격 섹션

```
구조:
  ┌─────────┐  ┌────────────┐  ┌─────────┐
  │ Free    │  │ Pro ⭐     │  │ Team    │
  │         │  │ (추천)     │  │         │
  │ $0/mo   │  │ $12/mo    │  │ $29/mo  │
  │         │  │           │  │         │
  │ 기능1 ✓ │  │ Free + ✓  │  │ Pro + ✓ │
  │ 기능2 ✓ │  │ 기능3 ✓   │  │ 기능5 ✓ │
  │ 기능3 ✗ │  │ 기능4 ✓   │  │ 기능6 ✓ │
  │         │  │           │  │         │
  │ [시작]   │  │ [시작] ★  │  │ [문의]  │
  └─────────┘  └────────────┘  └─────────┘

추천 플랜 강조:
  테두리: 2px primary
  배지: "Most Popular" 또는 "추천"
  높이: 다른 카드보다 8-16px 더 높게 (scale 1.02-1.05)
  또는: 배경색 변경 (primary/5%)

연/월 토글:
  위치: 가격 카드 위 중앙
  "월간" [토글] "연간 (20% 할인)"
  전환 시: 가격 숫자 애니메이션 (카운트 다운/업, 200ms)
  할인 표시: 원래 가격에 취소선 + 새 가격

기능 비교 테이블:
  플랜 3개 이상 시: 카드 아래에 상세 비교 표 추가
  행: 기능명 + 각 플랜별 ✓/✗/값
  sticky 헤더: 플랜명 + 가격 + CTA

가격 심리학:
  앵커링: 가장 비싼 플랜을 우측에 배치 → 중간 플랜이 합리적으로 보임
  디코이: 비슷한 가격의 열등 플랜 → 목표 플랜의 가치 부각
  무료 강조: "$0" 또는 "Free forever" (가입 장벽 제거)
  전체 가격 > 일일 가격: "$30/월" 대신 "하루 $1 미만" (비용 체감 축소)

  안티패턴: 가격을 숨기거나 "문의"로만 처리 (신뢰 감소)
  안티패턴: 플랜 차이가 불분명 ("Pro와 Team 차이가 뭐지?")
  안티패턴: 가격 페이지 도달 전 강제 가입
```

## FAQ 섹션

```
구조: 아코디언 (클릭 시 펼침)
  ┌────────────────────────────────────────────┐
  │  무료 체험은 얼마나 되나요?            [▾]   │
  ├────────────────────────────────────────────┤
  │  14일간 모든 기능을 무료로 사용할 수 있습니다. │
  │  신용카드가 필요 없습니다.                    │
  └────────────────────────────────────────────┘

항목 높이: 52-64px (닫힌 상태)
아이콘: ＋/－ 또는 ▸/▾, 우측 정렬, 회전 애니메이션 200ms
펼침 애니메이션: max-height 200-300ms ease-out
개수: 6-10개 (초과 시 카테고리 분류)
max-width: 720-800px (중앙 정렬, 너무 넓으면 가독성 저하)

질문 구성:
  가입/시작 관련: "무료인가요?", "해지 가능한가요?"
  기능 관련: "X를 할 수 있나요?", "Y와 연동되나요?"
  보안/신뢰: "데이터는 어디에 저장되나요?", "환불 정책은?"
  비교: "기존 도구와 뭐가 다른가요?"

  규칙: 질문 = 사용자의 언어, 답변 = 명확하고 짧게 (3줄 이내)
  규칙: 첫 항목을 가장 많이 묻는 질문으로 (자동 펼침 선택적)
  안티패턴: 답변에 마케팅 언어 과다 (FAQ는 솔직해야 신뢰를 준다)

SEO:
  FAQ에 FAQ Schema (JSON-LD) 추가 → 검색 결과에 풍부한 스니펫
```

## 최종 CTA 섹션

```
위치: 페이지 거의 끝, 푸터 바로 위
목적: 스크롤을 끝까지 한 방문자는 관심이 높다 → 강하게 전환 유도

구조:
  ┌────────────────────────────────────────────┐
  │            어두운 배경 또는 브랜드 색        │
  │                                            │
  │    "지금 시작하세요" (24-36px, Bold)         │
  │    부제목 한 줄 (16-18px, muted)            │
  │                                            │
  │    [주 CTA: 무료로 시작 →]                   │
  │                                            │
  │    "신용카드 불필요 · 14일 무료 체험"          │
  └────────────────────────────────────────────┘

규칙:
  히어로 CTA와 동일한 행동이어야 한다 (일관성)
  신뢰 보증 추가: "30일 환불 보장", "언제든 해지", "신용카드 불필요"
  배경: 나머지 페이지와 대비 (어두운 배경이 일반적)
  안티패턴: 여기서 새로운 정보를 소개하는 것 (요약/재확인만)
```
