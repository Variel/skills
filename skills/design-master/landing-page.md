# 랜딩 페이지 디자인 가이드

> 랜딩 페이지의 "좋은 디자인"은 목적에 따라 완전히 다른 형태를 띤다. 전환율이 전부인가, 브랜드 인상이 전부인가, 교육이 전부인가?

## 패러다임 A: 전환 엔진 — 모든 요소가 CTA를 향한다

"랜딩 페이지의 성공 = 전환율." **모든 섹션이 사용자를 하나의 행동(가입, 구매, 데모 요청)으로 밀어넣는다.**

핵심 사상: **페이지는 하나의 퍼널. 관심 → 신뢰 → 행동.**

### 설계 전 필수 확인(Hard Gate)

답이 명확하지 않으면 설계를 시작하지 말라:

- **제품**: 무엇인가, 어떤 문제를 해결하는가
- **대상**: 누구를 위한 것인가
- **목표**: 주된 전환, 보조 목표
- **가치 제안**: 무엇이 다른가/나은가
- **톤**: 브랜드 성격, 제약, 피할 단어
- **콘텐츠 제약**: 필수/금지 섹션, 법적 요구
- **시각 자료**: 브랜드 색상, UI 스크린샷

### 헤드라인 위계

| 수준 | 유형 | 예시 |
|------|------|------|
| 1 | **전환** | "드디어 받은편지함이 내 것처럼 느껴진다" |
| 2 | **결과** | "더 많은 콘텐츠를 배포하고, 더 많은 독자를 확보하세요" |
| 3 | **이점** | "10배 빠르게 글을 쓰세요" |
| 4 | **기능** | "AI 기반 글쓰기 어시스턴트" |

**전환이나 결과로 리드하라.** 이점/기능은 보조 카피에.

### 히어로 섹션 규칙

- **하나의 아이디어만.** 기능 목록 금지
- **헤드라인**: 핵심 약속/결과. 단독으로 의미 전달
- **서브라인**: 제품이 실제로 하는 일
- **CTA**: 주된 행동 하나. 선택적 보조 CTA
- **뷰포트**: 노트북 ~700px 높이 안에 핵심 전달
- **인지 한계**: 헤드라인, 서브라인, CTA, 하나의 비주얼, 신뢰 신호

### 페이지 구조 (SaaS 기준)

```
1. 헤더 — 로고, 내비게이션, 로그인, 주 CTA
2. 히어로 — 배지, 헤드라인, 서브라인, CTA, 비주얼, 신뢰 로고
3. 문제/해결 — "작동 방식" 단계 카드
4. 핵심 기능 — 3개 수직 스택 (헤드라인 + 설명 + 스크린샷)
5. 보조 기능 그리드 — 아이콘 카드
6. 사회적 증명 — 수치, 추천사
7. 가격 — 티어 + CTA
8. FAQ — 확장형 Q&A
9. 최종 CTA — 헤드라인 + CTA + 신뢰 보증
10. 푸터 — 로고, 내비게이션 칼럼, 저작권
```

### 섹션 리듬

- 텍스트/비주얼 **교대** — 텍스트만 연속 금지
- **어두운 섹션** → 신뢰감/깊이. **밝은 섹션** → 설명/디테일

## 패러다임 B: 내러티브 스크롤 — 스토리가 곧 설득

"사용자는 논리가 아니라 감정에 의해 움직인다." 전환율보다 **브랜드 인상과 감정적 연결** 우선.

핵심 사상: **페이지는 영화. 각 섹션은 장면, 스크롤은 시간의 흐름.**

```
참조: Apple 제품 페이지, Porsche, Nothing Phone

내러티브 구조 (3막):
  1막. 공감 — "당신의 고통을 안다" (Before 상태)
    → 풀블리드 이미지 + 짧은 질문형 헤드라인
    → 스크롤 트리거 애니메이션으로 감정 몰입

  2막. 전환 — "이렇게 바뀐다" (Bridge)
    → 제품 비주얼 점진적 공개 (스크롤에 따라 조립되는 UI)
    → 각 기능 = 하나의 풀스크린 섹션
    → 숫자는 카운트업 애니메이션

  3막. 도착 — "새로운 일상" (After)
    → 사용자 후기/사례 연구
    → 감성적 CTA ("시작하세요" 대신 "당신의 차례입니다")

스크롤 인터랙션:
  - Parallax (배경 0.5x, 전경 1x)
  - Scroll-triggered fade/slide-in
  - Sticky 섹션 (pin 후 내부 콘텐츠 교체)
  - 프로그레스 인디케이터
```

설계 규칙:
- **각 "장면"은 하나의 감정만 전달**
- CTA는 2-3곳, 이야기 흐름을 끊지 않게
- 모바일에서는 parallax/sticky 줄이거나 제거
- 이미지 lazy load + skeleton 필수

**적합**: 하드웨어, 럭셔리 브랜드, 크리에이티브 도구, 이벤트.

## 패러다임 C: 제품 주도(Product-Led) — 써보게 하라

설명 대신 **직접 체험**. "백 마디 말보다 한 번 써보는 게 낫다."

핵심 사상: **가입 전에 가치를 경험하게 하라. 사용자가 스스로 결론을 내리게.**

```
참조: Vercel, Supabase, Figma, Replit, Notion

페이지 구조:
  1. 히어로: 한 줄 설명 + 인라인 데모/플레이그라운드
  2. 인터랙티브 데모: 실제 UI 축소판, 클릭 가능 프로토타입
  3. 결과 증명: Before/After 코드 비교, 벤치마크
  4. 사용 사례: 실제 구현 사례 (코드 포함)
  5. 시작: "Get started free" — 신용카드 불필요 강조

핵심 요소:
  - 코드 블록이 1등 시민: syntax highlight, 복사/실행 버튼
  - 터미널 CTA: "npm install my-lib"
  - 비교 슬라이더: 좌(기존) / 우(제품)
  - 실시간 메트릭: "현재 N개 프로젝트에서 사용 중"
```

설계 규칙:
- **가입 벽을 최대한 늦춰라**: 체험 → 가치 확인 → 가입
- 데모는 실제 제품 UI 사용 (온보딩 비용 제로)
- 기술 문서 링크 숨기지 말라
- 데모 로딩 2초 이상이면 실패

**적합**: 개발자 도구, API, 노코드/로코드, 인프라, 오픈소스.

---

> **선택 기준**: "설명 듣고 결정" → 전환 엔진 / "감동받고 결정" → 내러티브 스크롤 / "직접 써보고 결정" → 제품 주도. 대부분은 세 가지를 비율을 달리해 혼합한다.

## 섹션별 상세 설계 패턴

"레이아웃 잡고, 색상 정하라"에서 멈추면 모든 랜딩 페이지가 동일해진다. 아래는 섹션마다 반복적으로 등장하는 결정들을 구체적으로 정리한 것이다.

### 헤더/내비게이션

```
기본 구조:
  ┌─────────────────────────────────────────────────┐
  │ [로고]   기능 | 가격 | 문서 | 블로그    [로그인] [CTA] │
  └─────────────────────────────────────────────────┘

높이: 56-72px
위치: sticky top (스크롤 시 고정)
배경: 투명 → 스크롤 후 blur(12px) + 배경색 (backdrop-filter)
z-index: 50 (모든 콘텐츠 위)
좌우 패딩: 24-48px (max-width와 동일)

네비게이션 항목:
  개수: 4-6개 (초과 시 "더보기" 드롭다운)
  폰트: 14-15px, weight 500
  활성: primary 색 또는 underline
  호버: opacity 0.7 → 1.0

CTA 버튼:
  헤더 내 CTA: 히어로 CTA와 동일한 행동 (일관성)
  크기: 다른 항목보다 시각적으로 돋보이게 (filled 버튼)

모바일:
  ┌──────────────────────┐
  │ [로고]        [☰] [CTA] │
  └──────────────────────┘

  햄버거 메뉴: 전체 화면 오버레이 또는 우측 슬라이드인
  CTA 버튼: 모바일에서도 헤더에 유지 (축약 가능: "시작하기" → 아이콘)
  안티패턴: 모바일에서 내비게이션을 숨기고 CTA도 숨기는 것

스크롤 반응:
  아래로 스크롤: 헤더 숨김 (transform: translateY(-100%))
  위로 스크롤: 헤더 재등장
  전환: 300ms ease-in-out
  대안: 항상 표시 (짧은 페이지) 또는 미니 헤더 (높이 축소)
```

### 히어로 섹션 변형

히어로는 랜딩 페이지의 "첫인상"이다. 방문자의 70%가 히어로만 보고 떠나거나 스크롤한다. **히어로에 모든 것을 거라.**

```
변형 A — 중앙 정렬 (Stripe, Linear):
  ┌────────────────────────────────────────────┐
  │              [배지: New]                     │
  │     가치 제안 헤드라인 (중앙)                  │
  │     부제목 설명 (중앙, max-width: 640px)      │
  │     [주 CTA]  [보조 CTA]                    │
  │                                            │
  │     [제품 스크린샷/비주얼]                     │
  │                                            │
  │     신뢰 로고 바                              │
  └────────────────────────────────────────────┘

  장점: 깔끔, 집중력 극대화
  단점: 비주얼이 약하면 밋밋
  적합: B2B SaaS, 개발자 도구, API

변형 B — 2칼럼 분할 (HubSpot, Shopify):
  ┌────────────────────┬───────────────────────┐
  │  배지               │                       │
  │  헤드라인            │   [제품 스크린샷]       │
  │  서브라인            │   또는 [비디오]         │
  │  [CTA]  [보조CTA]  │                       │
  │  신뢰 로고 바        │                       │
  └────────────────────┴───────────────────────┘

  비율: 텍스트 50-55% : 비주얼 45-50%, gap 48-64px
  장점: 텍스트 + 비주얼 동시 전달
  단점: 모바일에서 쌓이면 비주얼이 폴드 아래로
  적합: PLG 제품, 이커머스, 마케팅 도구

변형 C — 풀스크린 비주얼 (Apple, Nothing):
  ┌────────────────────────────────────────────┐
  │                                            │
  │        [배경: 풀블리드 이미지/비디오]          │
  │                                            │
  │            헤드라인 (오버레이)                 │
  │            [CTA]                            │
  │                                            │
  └────────────────────────────────────────────┘

  오버레이: gradient rgba(0,0,0,0.3-0.6)
  텍스트: 중앙 또는 좌하단
  장점: 감성적 임팩트 극대화
  단점: 정보 밀도 최소 (텍스트 2줄 한계)
  적합: 하드웨어, 럭셔리, 이벤트, 크리에이티브 도구

변형 D — 비디오 히어로:
  배경 비디오: 자동 재생, 무음, 루프
  포맷: MP4 (H.264), 해상도 1920×1080, 용량 5MB 이하
  대체: poster 이미지 (비디오 로드 전)
  모바일: 비디오 대신 정적 이미지 (데이터 절약)

  규칙: 비디오가 텍스트 가독성을 방해하면 안 됨 (오버레이 필수)
  규칙: 사용자가 비디오를 멈출 수 있어야 함 (접근성)

변형 E — 인터랙티브 히어로:
  타이핑 애니메이션 헤드라인 ("_를 위한 도구" → 순환)
  3D 모델/WebGL 배경 (Three.js)
  마우스 추적 파티클/그라디언트
  인라인 데모 위젯

  규칙: 인터랙션이 핵심 메시지를 가리면 안 됨
  규칙: prefers-reduced-motion에서 정적 버전 제공
  규칙: 모바일에서 성능 확인 필수 (GPU 가속 테스트)

히어로 배지(Announcement Badge):
  "✨ 새로운 기능 출시" / "Product Hunt #1" / "Y Combinator W24"
  크기: 12px/600, padding 4-6px 12-16px, radius 99px
  색상: 연한 primary 배경(primary/10%) + primary 텍스트
  위치: 헤드라인 바로 위, 12-16px 간격
  링크: 클릭 시 상세 페이지 또는 블로그로 이동
```

### 신뢰 & 사회적 증명

사회적 증명 없는 랜딩 페이지는 **"우리가 좋다고 하니까 좋다"**와 같다. 방문자는 당신의 말보다 다른 사용자의 경험을 믿는다.

```
로고 바 (Logo Cloud):
  위치: 히어로 바로 아래 또는 히어로 내부 하단
  레이블: "이 기업들이 사용합니다" 또는 "Trusted by"
  로고: 5-8개, 높이 24-32px, 회색조(grayscale)
  호버: 원래 색상으로 전환 (선택적)
  간격: 로고 간 32-48px
  정렬: 수평 중앙 또는 좌측 정렬
  모바일: 2행으로 축소하거나 수평 스크롤

  안티패턴: 알려지지 않은 로고만 나열 (신뢰 오히려 감소)
  안티패턴: 너무 많은 로고 (10개+ → 공간 낭비, 인지 과부하)

수치 증명 (Stats Bar):
  구조: 3-4개 수치 + 레이블을 수평 배치
  예시:
    10,000+        99.9%        <50ms        150+
    활성 사용자     가동률       응답 시간     국가
  수치: 32-48px, weight 700-800
  레이블: 14px, muted-foreground
  구분: 세로선 1px 또는 여백만으로 구분
  애니메이션: 뷰포트 진입 시 카운트업 (0 → 10,000+ / 1.5초)

  안티패턴: 무의미한 수치 ("1억 다운로드"가 개인 도구에서 무슨 의미인가?)
  안티패턴: 반올림이 과한 숫자 (523→500+ 는 OK, 12→10+는 인위적)

추천사(Testimonial):
  유형 A — 카드 그리드:
    ┌──────────────────┐
    │ "제품 덕분에..."  │
    │                  │
    │ 😊 이름, 직함     │
    │ @회사 로고        │
    └──────────────────┘
    2-3칼럼 그리드, 카드당 max 3줄 인용

  유형 B — 히어로 추천사:
    풀 너비, 큰 따옴표("), 인용문 20-28px italic
    하단: 사진(48-64px 원형) + 이름 + 직함

  유형 C — 비디오 추천사:
    가장 강력한 형태. 썸네일 + 재생 버튼
    길이: 30-90초. 짧을수록 좋다

  유형 D — 트위터/소셜 임베드:
    실제 트윗/포스트 스크린샷 또는 임베드
    신뢰도 높음 (수정 불가하므로)
    정리: masonry 그리드

  규칙: 이름 + 직함 + 회사 로고 + 얼굴 사진 = 최대 신뢰
  규칙: 사진 없는 추천사는 신뢰도 30% 하락
  규칙: 구체적인 결과("매출 23% 증가") > 모호한 칭찬("정말 좋습니다")

사례 연구(Case Study):
  구조: 로고 + 결과 수치(큰 폰트) + 한 줄 설명 + [사례 보기] 링크

### "작동 방식" (How It Works)

```
구조: 3-4단계 프로세스
  ① → ② → ③

표현:
  수평 카드: 3칼럼, 번호 + 아이콘 + 제목 + 설명
  수직 타임라인: 좌측 번호/아이콘, 우측 설명, 연결선
  일러스트: 각 단계가 하나의 이미지 scene

  번호: 48-64px, weight 700-900, primary 색 또는 muted
  아이콘: 번호 대신 사용 시 48px
  연결선: 번호 사이 1-2px dashed, muted 색

규칙:
  3단계가 이상적 (인지 부하 최소)
  각 단계: 동사로 시작 ("가입하세요", "설정하세요", "결과를 확인하세요")
  마지막 단계: 결과/성취 느낌 ("성장을 경험하세요")

  안티패턴: 5단계 이상 (3단계로 재구성하라)
  안티패턴: 실제 사용 흐름과 다른 이상화된 단계
```

## 마이크로카피

랜딩 페이지의 전환율은 디자인 50% + 카피 50%다. 작은 텍스트가 큰 차이를 만든다.

```
CTA 버튼 카피:
  나쁨: "제출", "확인", "시작"
  보통: "가입하기", "다운로드하기"
  좋음: "무료로 시작하기 →", "내 사이트 만들기 →", "14일 무료 체험"
  최고: 결과를 암시 ("성장 시작하기", "팀을 연결하세요")

  규칙: 동사 + 결과/혜택
  규칙: "→" 화살표는 진행감을 준다
  규칙: "무료" 또는 "신용카드 불필요"를 CTA 근처에

보조 텍스트 (CTA 아래):
  "30일 환불 보장" / "언제든 해지 가능" / "설정 5분이면 끝"
  폰트: 12-13px, muted, CTA 아래 8-12px

폼 필드:
  레이블: "이메일" → "업무용 이메일" (B2B일 때)
  플레이스홀더: "you@company.com"
  에러: "이메일을 입력해주세요" → "올바른 이메일 형식을 입력해주세요 (예: name@company.com)"

가격:
  "월 $12" → "$12/월, 연간 결제 시 $9/월"
  "무료 플랜" → "Free forever — 신용카드 불필요"
  취소선: "$29" → "$19" (할인 강조)

안티패턴:
  "여기를 클릭하세요" (모호, 접근성 나쁨)

## 성능 & Core Web Vitals

랜딩 페이지는 속도가 전환율이다. 로드 시간 1초 증가 = 전환율 7% 감소 (Google 데이터).

```
Core Web Vitals 목표:
  LCP (Largest Contentful Paint): <2.5초
    히어로 이미지/텍스트가 2.5초 안에 렌더
    → 히어로 이미지: preload + 적정 크기 + WebP/AVIF
    → 웹폰트: font-display: swap + preload
    → Critical CSS: above-the-fold CSS 인라인

  INP (Interaction to Next Paint): <200ms
    CTA 클릭 → 시각적 반응까지 200ms 이내
    → JavaScript 번들 최소화
    → 이벤트 핸들러 최적화

  CLS (Cumulative Layout Shift): <0.1
    콘텐츠 로딩 중 레이아웃 밀림 방지
    → 이미지/비디오: width + height 또는 aspect-ratio 명시
    → 웹폰트: size-adjust로 FOUT 최소화
    → 동적 콘텐츠: 자리 예약(placeholder) 후 교체

이미지 최적화:
  히어로: WebP/AVIF, 200-400KB 이하
  섹션 이미지: WebP, 100-200KB
  아이콘/로고: SVG 또는 아이콘 폰트
  지연 로딩: 폴드 아래 모든 이미지에 loading="lazy"
  반응형: srcset + sizes로 디바이스별 최적 크기

  <img srcset="img-480.webp 480w,
               img-960.webp 960w,
               img-1920.webp 1920w"
       sizes="(max-width: 768px) 100vw, 50vw"
       loading="lazy" alt="...">

폰트 최적화:
  폰트 수: 최대 2개 패밀리
  서브셋: 사용하는 글리프만 포함 (한글: unicode-range)
  preload: <link rel="preload" href="font.woff2" as="font" crossorigin>
  font-display: swap (FOIT 방지)
  가변 폰트(Variable Font): 하나의 파일로 여러 weight

JavaScript:
  초기 번들: 100KB 이하 (gzip)
  지연 로딩: 폴드 아래 인터랙션용 JS는 dynamic import
  써드파티: analytics, chat widget 등은 defer 또는 idle callback
  안티패턴: GTM 태그 20개+ (렌더링 블로킹)
```

## 모바일 반응형 전략

"데스크톱 먼저 만들고 모바일에 맞추기"는 가능하지만, **모바일에서 깨지는 요소를 방치하면 전환의 60%를 잃는다** (모바일 트래픽 비율 고려).

```
레이아웃 전환:
  데스크톱 2칼럼 → 모바일 1칼럼 스택
  데스크톱 카드 그리드 3열 → 모바일 1열 스택 또는 수평 스크롤
  데스크톱 사이드 이미지 → 모바일 이미지 위/텍스트 아래
  데스크톱 수치 4열 → 모바일 2×2 그리드

타이포 스케일:
  데스크톱 → 모바일 축소 비율:
  히어로 헤드라인: 56-72px → 32-40px
  섹션 헤드라인: 36-48px → 24-32px
  본문: 16-18px → 16px (축소하지 말라)
  CTA: 16px → 16px (축소하지 말라)

간격 축소:
  섹션 padding: 96-128px → 48-64px
  그룹 간격: 48-64px → 24-32px
  좌우 마진: 48px → 16-20px

CTA 모바일 최적화:
  전체 너비 버튼 (좌우 마진 16px)
  높이: 52-56px (터치 타겟)
  고정 하단 CTA (선택적): 스크롤 시 하단에 sticky CTA 바
    높이: 64-72px (safe area 포함)
    그림자: 0 -4px 12px rgba(0,0,0,0.1) (위쪽 그림자)

모바일 특화:
  비디오: 자동 재생 비활성화 또는 정적 이미지 대체
  parallax: 비활성화 (성능)
  호버 효과: 터치에서 의미 없음 → 제거 또는 탭으로 대체
  표: 수평 스크롤 또는 카드형으로 변환

  테스트: 실제 기기에서 확인 (에뮬레이터만으로 불충분)
```

## 업종별 랜딩 페이지 변형

모든 랜딩 페이지가 같은 구조일 수 없다. 업종에 따라 강조점이 다르다.

```
SaaS (B2B):
  강조: 데모/무료 체험 CTA + ROI 수치 + 기업 로고
  특이점: "Book a Demo" CTA, 사례 연구 중심, 보안 인증 뱃지
  히어로: 제품 스크린샷/대시보드 캡처
  사회적 증명: 기업 로고 + 사례 연구 + G2/Capterra 평점

SaaS (B2C):
  강조: 무료 가입 CTA + 기능 시연 + 사용자 후기
  특이점: 소셜 로그인, 가격 투명성, 커뮤니티
  히어로: 사용자 경험 비주얼 (사용 중 화면)
  사회적 증명: 사용자 수 + 평점 + SNS 언급

이커머스:
  강조: 제품 이미지 + 가격 + 구매 CTA + 리뷰
  특이점: 이미지 갤러리, 사이즈 가이드, 배송 정보
  히어로: 제품 이미지 풀 블리드 + 가격 + 즉시 구매
  사회적 증명: 별점 + 리뷰 수 + "N명이 지금 보고 있습니다"
  긴급성: "재고 3개 남음", "오늘 주문하면 내일 도착"

에이전시/포트폴리오:
  강조: 작업물 쇼케이스 + 프로세스 설명 + 문의 CTA
  특이점: 비주얼 중심, 케이스 스터디, 클라이언트 로고
  히어로: 대표 작업물 + 한 줄 포지셔닝
  CTA: "프로젝트 문의" 또는 "견적 요청"

이벤트/컨퍼런스:
  강조: 날짜/장소 + 스피커 + 티켓 CTA
  특이점: 카운트다운 타이머, 스케줄, 스피커 프로필
  히어로: 이벤트 비주얼 + 날짜 + 장소 + [티켓 구매]
  긴급성: "Early Bird 할인 D-7", "잔여 좌석 50석"
  사회적 증명: 이전 행사 사진/비디오 + 참가자 수

앱 다운로드:
  강조: 기기 목업 + 스토어 뱃지 + 기능 하이라이트
  히어로: 핸드폰 목업 (기울임 3D) + 앱스토어/구글플레이 버튼
  기능: 스크린샷 캐러셀 (3-5장)
  사회적 증명: 앱스토어 평점 + 리뷰 발췌
  안티패턴: QR코드만 크게 배치 (모바일 사용자는 QR을 못 스캔한다)

오픈소스:
  강조: GitHub 스타 수 + 코드 예시 + 빠른 시작 가이드
  히어로: 터미널 코드 블록 + "npm install" 또는 "pip install"
  기능: Before/After 코드 비교
  사회적 증명: GitHub 스타, 컨트리뷰터 수, 다운로드 수
  CTA: "Get Started" + "View on GitHub" (보조)
```

## 실무 수치 — 히어로, CTA, 섹션 간격

### 접근법 A: 넓은 여백 + 중앙 정렬 — 프리미엄 SaaS

Stripe, Linear 스타일.

```
히어로:
  전체 높이       = 100vh (최소 700px)
  콘텐츠 max-width = 800px, 중앙 정렬
  헤드라인         = 56-72px / weight 700 / tracking -0.03em / #111
  서브라인         = 18-20px / weight 400 / line-height 1.6 / #6B7280
  헤드-서브 간격   = 24px
  서브-CTA 간격    = 32px

CTA:
  주 CTA  = height 48px, padding 0 32px, font 16px/600
            radius 12px, 배경 #111, 글자 #FFF, hover → #333
  보조 CTA = height 48px, padding 0 24px, 테두리 1px #D1D5DB
  간격 = 12px

섹션:
  padding = 96-128px (상하)
  max-width = 1200px
  그룹 간 = 48-64px
  배경 교대: #FFF → #FAFAFA → #FFF → #111
```

**적합**: B2B SaaS, 핀테크, API, 개발자 도구.

### 접근법 B: 밀도 높은 정보 — 전환율 최적화

HubSpot, Shopify 스타일.

```
히어로:
  높이 = auto (500-600px)
  2칼럼 = 텍스트 55% : 비주얼 45%, gap 48px
  헤드라인 = 40-52px / weight 800 / line-height 1.15
  배지 = 12px/600, padding 4px 12px, radius 99px, #EEF2FF/#4338CA
  신뢰 로고 바 = CTA 아래 16px, 로고 높이 24-32px, 회색조

CTA:
  주 CTA  = height 52px, padding 0 28px, font 16px/700
            radius 8px, 그라디언트 (#7C3AED → #6D28D9)
            box-shadow: 0 4px 14px rgba(124,58,237,0.35)

섹션:
  padding = 64-80px (상하)
  3칼럼 카드 = padding 32px, gap 24px, radius 16px
```

**적합**: PLG 제품, 무료 체험, 스타트업 런칭.

### 접근법 C: 풀스크린 비주얼 — 감성 우선

Apple, Arc Browser 스타일.

```
히어로:
  높이 = 100vh, 배경: 풀블리드 이미지/비디오
  오버레이 = linear-gradient(rgba(0,0,0,0.4), rgba(0,0,0,0.6))
  헤드라인 = 64-96px / weight 700 / #FFFFFF / 중앙
  CTA = height 56px, padding 0 40px, 배경 #FFF, 글자 #111, radius 99px
        hover → scale(1.03)

섹션:
  비주얼 = 100vh, 배경 이미지/비디오, 텍스트 오버레이
  텍스트 = padding 120-160px, max-width 640px, font 24-28px/300

색상 (다크 기조):
  배경 #000000, 텍스트 #FFFFFF/#rgba(255,255,255,0.6)
  강조 단일 컬러 (예: #FF6B35)
```

**적합**: 하드웨어, 럭셔리, 크리에이티브 도구, 이벤트.

### 접근법 D: 인터랙티브 데모 중심

Vercel, Supabase 스타일.

```
히어로:
  2칼럼: 텍스트 40% + 코드 60%
  헤드라인 = 48-56px / weight 700
  코드 블록 = padding 24px, 배경 #0D1117, radius 12px
              font 14px JetBrains Mono, line-height 1.7
              syntax: keyword #FF7B72, string #A5D6FF, function #D2A8FF

CTA:
  주 = "Get started →" height 44px, 배경 #FFF, 글자 #000, radius 8px
  터미널 CTA = height 44px, 배경 #1E1E1E, 글자 #4ADE80, 복사 아이콘

기능 섹션:
  탭 데모 = 탭 height 40px, fade 200ms
  비교 = Before(#FEE2E2) vs After(#DCFCE7)
```

**적합**: 개발자 도구, API, 인프라, 오픈소스.

## 스크롤 인터랙션 상세

```
Intersection Observer 패턴:
  요소가 뷰포트에 진입 시 애니메이션 트리거
  threshold: 0.2 (20% 보일 때)
  rootMargin: "0px 0px -10% 0px" (약간 더 스크롤해야 트리거)

  기본 등장: opacity 0 → 1, translateY(20px) → 0, 600ms ease-out
  엇갈림(stagger): 카드 그리드에서 각 카드 100-150ms 간격으로
  한 번만: 한번 등장하면 다시 사라지지 않음 (once: true)

Sticky 섹션 (Pin and Swap):
  좌측 텍스트 고정 + 우측 비주얼 스크롤에 따라 전환
  구현: position: sticky + Intersection Observer로 활성 항목 변경
  높이: 각 단계 × 100vh (3단계면 300vh 섹션)

  규칙: 모바일에서는 sticky 제거 → 일반 스택 레이아웃
  규칙: 현재 활성 항목을 시각적으로 강조 (opacity, color, underline)

카운트업 애니메이션:
  트리거: 뷰포트 진입 시
  기간: 1.5-2초
  이징: ease-out (빠르게 시작, 천천히 끝)
  포맷: 쉼표 구분 (1,234,567) 또는 축약 (1.2M)
  소수점: 필요 시 마지막 0.3초에 소수점 이하 추가

Parallax:
  배경 이미지: transform: translateY(calc(var(--scroll) * 0.3))
  전경 요소: 기본 스크롤 (1x)
  규칙: 3개 이상 레이어 금지 (혼란)
  규칙: 텍스트에 parallax 적용하지 말라 (가독성 저하)
  모바일: 완전 비활성화 (성능, 이질감)

  prefers-reduced-motion:
    모든 스크롤 애니메이션 → 즉시 표시 (opacity: 1, transform: none)
    parallax → 정적 배치
    카운트업 → 최종 값 즉시 표시
```

## SEO & 메타데이터

```
필수 메타:
  <title>: 60자 이내, 핵심 키워드 포함
  <meta name="description">: 155자 이내, CTA 포함
  Open Graph: og:title, og:description, og:image (1200×630px)
  Twitter Card: twitter:card, twitter:title, twitter:image

구조화된 데이터:
  Organization: 회사 정보 (logo, name, url)
  Product: 가격, 평점 (이커머스)
  FAQ: FAQ 섹션의 질문/답변 (FAQ Schema)
  Breadcrumb: 내비게이션 경로

시맨틱 HTML:
  <header>: 헤더/내비게이션
  <main>: 본문
  <section>: 각 섹션 (aria-labelledby로 제목 연결)
  <article>: 블로그 글, 사례 연구
  <footer>: 푸터

  h1: 페이지당 하나 (히어로 헤드라인)
  h2: 각 섹션 제목
  h3: 섹션 내 하위 제목

  안티패턴: 시각적 크기만으로 헤딩 수준 결정 (h2 건너뛰고 h4)

이미지 SEO:
  alt 텍스트: 모든 의미 있는 이미지에 설명
  장식 이미지: alt="" (빈 문자열)
  파일명: "hero-dashboard-preview.webp" (의미 있게)

성능 = SEO:
  Core Web Vitals → Google 검색 순위 팩터
  모바일 친화성 → Google 모바일 우선 인덱싱
  HTTPS → 필수
```
