# 시각적 완성도 + 검증 + 안티패턴

## Part VI. 시각적 완성도 — 디테일의 기술

### 1. 미학적 방향의 의도성

디자인 시작 전에 **대담한 미학적 방향**을 선택하고 정밀하게 실행하라. 의도성이 강도보다 중요하다.

가능한 방향들:
- 극도의 미니멀리즘
- 맥시멀리스트
- 레트로-퓨처리스틱
- 유기적/자연적
- 럭셔리/세련됨
- 놀이적/장난감 같은
- 에디토리얼/매거진
- 브루탈리스트/날것
- 아르데코/기하학
- 소프트/파스텔
- 산업적/실용적

**핵심은 의도성(intentionality)이지 강도(intensity)가 아니다.** 선택한 방향을 끝까지 밀고 나가라.

### 2. 타이포그래피

- **독특하고 아름다운 폰트를 선택하라.** Arial, Inter, Roboto 같은 범용 폰트를 피하라
- 특징적인 디스플레이 폰트 + 세련된 본문 폰트 조합
- 매번 다른 조합 시도 — 같은 폰트에 수렴하지 말라

### 3. 색상과 테마

- CSS 변수로 일관성 확보
- **지배적 색상 + 날카로운 강조색** > 소심하게 균등 분배된 팔레트
- 시맨틱 색상을 목적에 맞게 (에러=에러색, 성공=성공색)
- 하드코딩된 hex/rgb 대신 변수/토큰 사용

#### 디자인 토큰 참고

| 토큰 | 용도 |
|------|------|
| `background` | 페이지 배경 |
| `foreground` | 주 텍스트 |
| `muted-foreground` | 보조 텍스트, 플레이스홀더 |
| `card` | 카드 배경 |
| `border` | 테두리, 구분선 |
| `primary` | 주 행동, 브랜드 |
| `destructive` | 위험 행동 |

### 4. 모션과 인터랙션

- 잘 짜인 페이지 로드 애니메이션(엇갈린 등장) > 흩어진 마이크로인터랙션
- CSS-only 우선. React에서는 Motion 라이브러리
- 스크롤 트리거와 호버에 놀라움
- **맥시멀 → 정교한 애니메이션. 미니멀 → 절제된 정밀함.**

### 5. 공간과 배경

- 단색 배경에 안주하지 말라 — **분위기를 만들어라**
- 그라디언트 메시, 노이즈 텍스처, 기하학 패턴, 그레인 오버레이
- 비대칭, 겹침, 대각선 흐름, 그리드 깨기

### 6. 이미지 사용 위계

| 순위 | 유형 | 설명 |
|------|------|------|
| 1 | **전환 이미지** | After 상태의 사람들. 제품은 없거나 주변부 |
| 2 | **맥락적 사용** | 실제 환경에서 제품 사용하는 사람들 |
| 3 | **환경 속 제품** | 사용/결과를 암시하는 환경에 놓인 제품 |
| 4 | **고립된 제품** | 제품 단독. 최소한으로 |

**모든 이미지는 방문자의 미래 삶에서 온 한 장면이어야 한다.**

## Part VII. 검증 — 디자인이 제대로 작동하는가

### 1. 자기 점검 체크리스트

#### 사용자 관점

- [ ] 처음 방문한 사용자가 3초 안에 화면 목적을 이해할 수 있는가?
- [ ] 다음에 해야 할 행동이 명백한가?
- [ ] 사용자가 추측해야 하는 부분이 있는가? → 재설계
- [ ] 모든 상태(로딩, 빈, 에러, 성공)가 고려되었는가?
- [ ] 모든 행동에 피드백이 있는가?

#### 구조 관점

- [ ] 화면에 하나의 지배적 목적이 있는가?
- [ ] 하나의 지배적 시각 영역이 있는가?
- [ ] 정보 위계가 명확한가?
- [ ] 밀도가 의도적인가?
- [ ] 불필요한 요소가 있지는 않은가?

#### 일관성 관점

- [ ] 같은 문제를 같은 방식으로 해결하고 있는가?
- [ ] 간격이 일관된 척도를 따르는가?
- [ ] 색상이 시맨틱 토큰을 사용하는가?
- [ ] 타이포그래피 체계가 일관적인가?

#### 확장성 관점

- [ ] 데이터가 10배 늘어나도 깨지지 않는가?
- [ ] 항목이 추가되어도 구조가 유지되는가?
- [ ] 다른 브레이크포인트에서도 위계가 살아있는가?

### 2. 콘텐츠 우선 검증

시각 디자인을 제거하고 콘텐츠만 남겼을 때:
- 가치가 몇 초 안에 명확한가?
- 흐름이 논리적인가?
- 이점이 결과 중심인가?
- 신뢰가 충분한가?
- **비주얼 없이도 페이지가 작동하는가?**

콘텐츠가 이 테스트를 통과해야 비주얼 작업을 시작할 수 있다.

### 3. 주기적 시각 검증

디자인 작업 중 주기적으로 스크린샷 확인:
- 생성된 결과가 요구사항을 충족하는가?
- 시각적 오류, 결함, 정렬 불일치가 없는가?
- 의도한 레이아웃과 일치하는가?
- 컴포넌트가 올바르게 구성되었는가?
